Before you begin

If Linux is your first 4finance workstation and you never used Windows, you must first log in to Windows domain (using your domain password) at least once, e.g. from your colleagues' computer. You will then be asked to change your password. Otherwise some services like e-mail or terminal server via Remmina won't accept your initial password.

Disk Encryption

During Linux installation Full disk encryption should be configured. Before installing Linux check Linux documentation how to do it. By default during installation full disk encryption is not enabled  (Arch, Manjaro, Ubuntu, Mint)  and after installation complete it can be very hard or impossible to enable it. 

Dock support

  • HP Zbook Studio G3 + HP Elite Thunderbolt 3 Dock
    • Disable Thunderbolt security in BIOS (Tested with kernel 2.6)
    • (Ubuntu 16.04) In case of 3+ monitors are not working properly increasing video memory in BIOS might help

Network connection

4Finance is using Checkpoint Identity Agent (IA) as means to authorize user access to corporate network. IA is automatically deployed to all 4Finance Windows computers. Linux is not supported by agent so it cannot be installed, therefore Linux station is being treated as untrusted device from Checkpoint gateway perspective.

Since Linux devices don't have IA they can't access corporate resources and are allowed to access public network (open ports are limited).

Trusted root certificate

Due to request for web traffic filtering and monitoring in several offices across group, it is suggested to add this certificate (attached below) to trusted root certificate to all unmanaged machines to be able to browse internet without annoying security popups. For now this feature is only in some offices, but later it will be implemented in all offices.

checkpoint.ofg.local.cer


What options do I have to connect physically?

  • Cable LAN
    • You can use your ethernet cable. Our systems will automatically detect that you are not using Windows and will not allow to access internal resources (excluding some, like Jira, Wiki).
  • 4Finance WIFI (this connection option soon will be no longer available)

    • You can use internal WIFI. Our systems will automatically detect that you are not using Windows and will not allow to access internal resources (excluding some, like Jira, Wiki).
      By default this connection is hidden and allows to connect only from computers having 4Finance certificates.

      To be able to connect:

      1. Infra team needs to add your user to "Wifi access for non domain computers" group in AD
      2. In your workstation choose "Connect to Hidden Wi-Fi Network"
        1. Connection: New...
        2. Network Name: 4finance WiFi    (case-sensitive!)
        3. Wi-Fi security: WPA & WPA2 Enterprise
        4. Authentication: Protected EAP (PEAP)
        5. Anonymous identity: <leave blank>
        6. CA certificate: none
        7. PEAP version: Automatic
        8. Inner Authentication: MSCHAPv2
        9. Username: <your domain user> (i.e. maris.mekss)
        10. Password: <your domain account password>
      3. It will alert you that no CA certificate has been chosen. Ignore this
      4. It will ask to input your Linux user password. 
  • 4Finance Public WIFI

    • Completely public wifi with only public access. Used mostly for guests. Not recommended as you won't be able to resolve internal DNS names.

    • The password is changing monthly!
    • Right now, new passwords are published on Intranet page. Here's the announcement about the new one (05-10-2018): cHekof8vub.
  • 4Finance VPN
    • Pulse Secure VPN


Recommendation:

We strongly suggest using cable or 4Finance Wifi connection as these connections provide you internal DNS and access to some resources, like JIRA and Wiki.

There is a problem with Thunderbird calendar events that are keep appearing, disable it in the config by setting false to calendar.alarms.showmissed.
- should be fixed in the most recent version of Thunderbird / Lighting - though sometimes still happens and a restart of Thunderbird together with a manual kill (of Thunderbird processes) from the terminal is required to get rid of the popup

Adding a publicly shared calendar (rooms) in Thunderbird

  1. Install the following Add-ons for Thunderbird:
    1. Exchange EWS Provider (link)
      1. download xpi file and install Add-on from file in Thunderbird
    2. Lightning (Thunderbird > Tools > Add-ons > Search)
  2. Thunderbird > Calendar > Right click > New Calendar > On the network > Next
  3. Microsoft Exchange 2007 / 2010 / 2013 > Next
  4. Name: <Local name of the shared Calendar>
    Email: None (Important to not select your email address) > Next
  5. Uncheck Use Exchange's autodiscovery
    Server URL: https://mail.4finance.com/ews/exchange.asmx
    Primary email address: <the room email address> (e.g. CZ-IT-Meeting.Room.2@ofg.local - which is CZ IT Staromak)
    Username: <your domain user name> 
    Domain name: leave blank
    Shared Folder Id: leave blank
    Click "Check Server and mailbox"
  6. Next > Finish > Enjoy

Opening attached e-mails (.eml)

Thunderbird can't open .eml files attached to e-mails. A quick workaround is to save such file and then choose it with File -> Open in Thunderbird. This time Thunderbird opens such e-mail properly.

Troubleshooting

No connection to ofg.local hosts

Probably avahi daemon is running on your PC. Try do change domain-name from .local to e.g. .alocal in /etc/avahi/avahi-daemon.conf:

?
[server]
domain-name=.alocal

and restart avahi daemon:

?
$ sudo service avahi-daemon restart

After that you can ping server and have access to them: 

 ping dcm1-tst-002.ofg.local

 PING dcm1-tst-002.ofg.local (192.168.73.14) 56(84) bytes of data.

 64 bytes from 192.168.73.14: icmp_seq=1 ttl=63 time=61.6 ms

More information is available here:

http://www.lowlevelmanager.com/2011/09/fix-linux-dns-issues-with-local.html

JIRA/Stash doesn't respond

If all other services work fine, but you have problems with JIRA or Stash (it times out etc.), it may be because jira.4finance.net is resolved to a different IP publicly than in the 4finance network. So if in your DNS settings you have both 4finance DNS server (192.168.49.251) and public DNS servers (e.g. Google's 8.8.8.8, 8.8.4.4), sometimes it's resolved to the public one, which is not right.

On Mac OS X, you can force a domain to be resolved with specific DNS server. Create a file named /etc/resolver/jira.4finance.net (or for stash respectively) specifying the 4finance DNS server:

?
nameserver 192.168.49.251

And from now on, the service will always be resolved to the correct IP.

Mint: edit /etc/resolvconf/resolv.conf.d/head file and add following lines, then run sudo resolvconf -u:

/etc/resolvconf/resolv.conf.d/head
?
nameserver 192.168.49.251
nameserver 192.168.50.252
nameserver 192.168.72.251
nameserver 8.8.8.8


Can't login to Terminal, SNX, OWA

It might happen that suddenly you won't be able to log into any of the mentioned services. It's quite possible that's because your domain password has expired. Normally you would be notified few times about that fact via e-mail, but it might be that, due to a human error, you won't be in a proper user's group, so such e-mails won't be even sent to you.

To be able to work in a normal fashion, you need to reset your password, which can be done when logging into your domain account - for example from your colleague laptop that's running on Windows. Simply log in to the domain with your current password and Windows will ask you to change your password to a new one.
Second thing is to inform someone that you've not received any warning about the fact that your password will expire (by using zendesk or skype). Normally it should be possible to reset your password through owa.


Fail to start VirtualBox related services

On freshly installed Linux machine (experienced on Fedora 28, but internet says also seen on Ubuntu), with UEFI Secure boot enabled, might be that there will be error regarding vboxdrv kernel module, which is missing or can’t be installed.

If so, ensure that kernel-devel package is installed, switch to root user and go to root's home directory.

Generate signing keys
?
openssl req -new -x509 -newkey rsa:2048 -keyout MOK.priv -outform DER -out MOK.der -days 36500 -subj "/CN=VirtualBox modules/"

Export key passphrase which has been provided while generating signing key

Export key passphrase
?
export KBUILD_SIGN_PIN='your password from previous step'

Sign module (vboxdrv for this example, but repeat for other modules from following command output for full functionality: ls $(dirname $(modinfo -n vboxdrv))/vbox*.ko

Signing module
?
/usr/src/kernels/$(uname -r)/scripts/sign-file sha256 ./MOK.priv ./MOK.der $(modinfo -n vboxdrv)

Confirm the module is signed

Confirm the module is signed
?
tail $(modinfo -n vboxdrv) | grep "Module signature appended"

Register the keys to Secure Boot

Register keys to Secure Boot
?
mokutil --import MOK.der


Reboot computer to Enroll MOK (Machine Owner Key), during computer start - ensure you use computers built in keyboard as devices attached to dock might not work properly. You will have a few seconds to pres any key when blue screen with title Perform MOK management is prompting for that, otherwise computer will boot up and changes won't be applied. Then procedure should be restarted, except generating signing keys.

Steps during Perform MOK management dialogue (might differ a bit, but should be +- the same): Enroll MOK → Continue → Yes → Enter password you used to generate signing keys → OK

Computer will be rebooted and afterwards you should be able to start VirtualBox instances.
